# Product Guidelines: FoodPal

## 1. Visual Design Principles
- **Minimalist Aesthetic:** Focus on whitespace and high-quality typography. Avoid visual clutter.
- **Rounded Corners:** Use a consistent corner radius of 24.0 or higher for cards, buttons, and input fields to maintain a soft, modern feel.
- **Soft Shadows:** Employ subtle elevation to create depth without harsh borders.
- **Consistent Spacing:** Adhere to an 8px or 16px grid system to ensure balanced layouts.

## 2. User Experience (UX)
- **Direct Action:** Primary actions (like logging food) should be accessible with a single tap.
- **Implicit Feedback:** Use subtle haptics and micro-animations to confirm user actions.
- **Hidden Complexity:** Progressive disclosure should be used. Advanced data and settings should be tucked away until explicitly requested by the user.

## 3. Tone and Copy
- **Professional but Approachable:** The AI should sound like a knowledgeable, supportive peer, not a clinical data processor.
- **Concise Instructions:** Keep labels and help text short. The UI should be intuitive enough to require minimal explanation.
- **Action-Oriented:** Use active verbs (e.g., "Log Breakfast" instead of "Breakfast Logging").

## 4. Mobile-First Consistency
- **Touch Targets:** Ensure all buttons and interactive elements meet standard accessibility sizes (min 44x44 pixels).
- **Responsive Layouts:** Components must scale gracefully between different screen sizes while maintaining the core "To-Do List" card layout.
